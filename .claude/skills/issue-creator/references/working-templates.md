# Working Documents テンプレート

## context.md

コンテキスト復元が主目的。スペックを複製せず、参照リンク + Issue固有の補足に留める。

```markdown
# コンテキスト: [タイトル]

Issue: #XX

## スペック参照
- [スペック](../../specs/YYYY-MM-DD-xxx.md)（該当セクション名を明記）
- [実装プラン](../../plans/YYYY-MM-DD-xxx.md)（あれば）
- [ADR-XXXX](../../../adr/XXXX-xxx.md)（関連する設計判断があれば）

## このIssueで実現すること
[スペックの中でこのIssueがカバーする範囲を1〜3文で]

## このIssue固有の補足
[スペックに書かれていないが、ブレインストーミングで決まったこと・制約・注意点]

## スコープ外
- 今回は対応しないこと
```

## tasklist.md

```markdown
# タスクリスト: [タイトル]

Issue: #XX

## 作業内容
- [ ] タスク1
- [ ] タスク2
- [ ] タスク3

## 完了条件
- 条件1
- 条件2
```
