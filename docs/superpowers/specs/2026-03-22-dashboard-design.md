# ダッシュボード（リニューアル + ナビゲーション）設計スペック

## 1. 概要

ダッシュボードページ（`/dashboard`）のリニューアルと、全ページ共通ナビゲーションバーの新規実装。

### 1.1 目的

- ユーザーが進行中の作品を一覧し、ワンクリックで進捗を更新できるようにする
- アプリ全体の共通ナビゲーションを導入し、ページ間の移動を直感的にする

### 1.2 スコープ（フェーズ1ミニマル）

- 視聴中リスト表示（`status=watching` の全記録）
- ジャンル別クイックアクションボタン（+1話、観た、クリア等）
- グローバルナビゲーションバー（全ページ共通）

### 1.3 スコープ外

- 最近の記録セクション（フェーズ2で検討）
- 統計サマリー（フェーズ2で検討）
- おすすめセクション（フェーズ4）

---

## 2. ナビゲーションバー

### 2.1 配置

- **トップバー（横並び）**: ロゴ左、ナビ項目右
- App.tsxレベルで認証済みページに共通表示
- 高さは最小限（1行に収まる構成）

### 2.2 ナビ項目

| 項目 | パス | 状態 |
|------|------|------|
| ホーム | `/dashboard` | 有効 |
| 検索 | `/search` | 有効 |
| ライブラリ | `/library` | 有効 |
| コミュニティ | — | グレーアウト（未実装） |
| おすすめ | — | グレーアウト（未実装） |
| マイページ | — | グレーアウト（未実装） |

- 現在のページに対応するナビ項目にはアクティブ状態（下線）を表示
- グレーアウト項目はクリック不可

### 2.3 ユーザーメニュー

- ナビバー右端にイニシャルアバター（丸型、背景 `#2c2c2c`、白文字）
- クリックでドロップダウンメニューを表示:
  - ユーザー名（太字）
  - メールアドレス（グレー）
  - 区切り線
  - マイページ（グレーアウト、「準備中」表記）
  - 区切り線
  - ログアウト
- ドロップダウン外クリックで閉じる

### 2.4 デザイン仕様

- ロゴ: `Fraunces`（セリフ体）、22px、太字
- ナビ項目: 13px、600weight、letter-spacing 1.5px、uppercase
- 下部ボーダー: `2px solid #2c2c2c`（Recollyのデザイントークンに準拠）
- アバター: 32px × 32px、`border-radius: 50%`

## 3. ダッシュボード — 視聴中リスト

### 3.1 セクションタイトル

- 「進行中」（SectionTitleコンポーネントを使用）
- 全ジャンルを包含する汎用的な名称

### 3.2 データ取得

- 既存の Records API を使用: `GET /api/v1/records?status=watching`
- ソート: 更新日降順（最近更新した作品が上に来る）

### 3.3 各行の構成

```
[カバー画像] [作品タイトル]              [アクションボタン]
             [ジャンルラベル（ジャンルカラー）]
             [進捗テキスト]
```

- カバー画像: 40px × 56px、border-radius 3px
- 作品タイトル: 15px、600weight
- ジャンルラベル: 10px、600weight、uppercase、ジャンルカラーで表示
- 進捗テキスト: 13px、`#6b6b6b`
- 行全体: `border: 1px solid #e0e0e0`、border-radius 6px、ホバーで背景色変化
- 行クリック: `/works/:id` へ遷移

### 3.4 進捗テキスト表示

| ジャンル | 話数あり | 話数なし |
|---------|---------|---------|
| アニメ | `12 / 25話` | — |
| ドラマ | `3 / 62話` | — |
| 漫画 | `89 / 108巻` | — |
| 本 | `読書中` | `読書中` |
| 映画 | `—` | `—` |
| ゲーム | `プレイ中` | `プレイ中` |

### 3.5 アクションボタン（ジャンル別）

| ジャンル | ラベル | 動作 |
|---------|--------|------|
| アニメ | +1話 | `current_episode` を +1 |
| ドラマ | +1話 | `current_episode` を +1 |
| 漫画 | +1巻 | `current_episode` を +1 |
| 本 | 読了 | `status` → `completed` |
| 映画 | 観た | `status` → `completed` |
| ゲーム | クリア | `status` → `completed` |

#### ボタン仕様

- 横幅: 72px固定（全ジャンル統一）
- スタイル: 背景 `var(--color-text)`、白文字、border-radius 4px（実装時はtokens.cssの変数を使用）
- クリック時: 画面遷移なし。APIで更新し、リストを再レンダリング
- 話数ありメディアで最終話到達時: 既存の自動完了ロジック（`status` → `completed`）が発動し、リストから消える
- 話数なしメディア（本・映画・ゲーム）: ボタン押下で `status` → `completed` に変更、リストから消える

## 4. 空状態（進行中の作品が0件）

### 4.1 デザインパターン: B3（ジャンルカード + ステップガイド）

#### 構成（上から順）

1. **見出し**: 「はじめましょう」（18px、600weight、中央揃え）
2. **ジャンルピル**: 6ジャンルを横並びで中央揃え
   - 各ピル: ジャンルドット + ジャンル名、角丸20px
   - 背景: ジャンルカラーの10%透過、ボーダー: ジャンルカラーの30%透過
3. **説明文**: 「すべてのジャンルをまとめて記録・管理できます」（14px、`#6b6b6b`）
4. **3ステップガイド**（横並び、中央揃え）
   - ステップ1: 「作品を探す」— 検索ページで気になる作品を見つける
   - ステップ2: 「記録する」— ステータスを選んでライブラリに追加
   - ステップ3: 「進捗を更新」— ワンクリックで「+1話」
   - 各ステップ: 丸囲み番号（40px、`border: 2px solid #2c2c2c`）
5. **CTAボタン**: 「作品を探す」→ `/search` へ遷移（btn-primary スタイル）

### 4.2 コンテナ

- `border: 1px dashed #e0e0e0`、border-radius 8px
- padding: 48px

## 6. API

### 6.1 使用するAPI（既存）

- `GET /api/v1/records?status=watching` — 進行中リスト取得（sortパラメータ省略 = デフォルトの `updated_at` 降順）
- `PATCH /api/v1/records/:id` — 進捗更新（`current_episode` または `status` の更新）

### 6.2 新規APIエンドポイント

なし。既存APIで全て対応可能。

## 7. インタラクション詳細

### 7.1 +1話ボタン押下時のフロー

1. ボタンクリック
2. 楽観的UIで `current_episode` を +1 して即座に表示更新
3. `PATCH /api/v1/records/:id` で `current_episode` を更新
4. `current_episode === total_episodes` の場合、既存の自動完了ロジックが発動
5. 完了した記録はリストから消える
6. API失敗時はロールバックしてエラー表示

### 7.2 完了ボタン（観た / 読了 / クリア）押下時のフロー

1. ボタンクリック
2. `PATCH /api/v1/records/:id` で `status` を `completed` に更新
3. 成功したら記録をリストから除去（アニメーション不要、シンプルに消える）
4. API失敗時はエラー表示

### 7.3 ドロップダウンメニュー

- アバタークリックで開く
- メニュー外クリックまたはEscキーで閉じる
- ログアウト: 既存の `useAuth().logout()` を呼び出し、`/login` へリダイレクト
