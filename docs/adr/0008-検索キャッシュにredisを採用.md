# ADR-0008: 検索キャッシュにRedisを採用

## ステータス
承認済み

## 背景
作品検索機能では、外部APIの検索結果をWorksテーブルに直接保存せず、ユーザーがライブラリに追加した時点でDB保存する方針とした。
検索結果を一時的に保持するキャッシュ層が必要。

## 選択肢
1. **Redis** — メモリ上にデータを保存する高速データベース。Docker Composeにサービスとして追加
2. **Rails.cache（メモリストア）** — Railsアプリのプロセスメモリ内にキャッシュ

## 決定
**Redis**を採用する。

## 理由
- TTL（自動期限切れ）機能が検索結果の一時キャッシュに適している
- プロセス再起動でもキャッシュが失われない
- 将来のフェーズ4（レコメンド機能）等でもRedisが必要になる可能性が高い
- Docker Compose環境なのでサービス追加が容易
