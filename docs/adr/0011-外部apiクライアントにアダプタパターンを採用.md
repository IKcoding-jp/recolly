# ADR-0011: 外部APIクライアントにアダプタパターンを採用

## ステータス
承認済み

## 背景
4つの外部API（TMDB, AniList, Google Books, IGDB）のクライアントをどう設計するか。
検索機能では「ジャンル指定検索」と「全ジャンル横断検索」の両方をサポートする必要がある。

## 選択肢
1. **アダプタパターン** — 共通インターフェース（`search(query)`等）を定義し、各API用クラスが実装
2. **個別サービスクラス** — 各APIごとに独立したサービスクラスを作り、コントローラーで使い分け

## 決定
**アダプタパターン**を採用する。

## 理由
- コントローラーは「どのAPIか」を意識せず `client.search(query)` と統一的に呼べる
- 横断検索時に全アダプタを回すだけで実現できる
- 新しい外部API追加時もアダプタクラスを1つ追加するだけ
- 各APIのレスポンス形式の違いをアダプタ内部で吸収し、統一フォーマットで返す
